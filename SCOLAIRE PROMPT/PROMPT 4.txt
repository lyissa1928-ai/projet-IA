1) Modèle de données (Prisma)
1.1 CourseUnit (UE)

Champs :

id (uuid)

code (string, unique global ou unique par programme — choisir global unique)

title

programId (FK Program)

levelId (FK Level)

semesterId (FK Semester)

credits (int > 0)

coefficient (float > 0)

vh_cm (int >= 0)

vh_td (int >= 0)

vh_tp (int >= 0)

isMutualized (bool, default false)

status enum: ACTIVE | INACTIVE

createdAt, updatedAt

Contraintes

(programId, levelId, semesterId, code) cohérent

credits > 0

coefficient > 0

total heures >= 0

level.programId == programId

semester doit appartenir à année active (option warning)

2) Validation Cohérence Crédits (Business Logic)
2.1 Règle Semestre

Total crédits par :

(programId + levelId + semesterId)

doit respecter cohérence académique.

Par défaut :

30 crédits par semestre (configurable plus tard)

Implémenter validation :

Lors création/modification UE :

recalcul total crédits semestre

si dépassement seuil configurable (ex: 30 ± 10%) → warning

si strict mode activé → blocage

Ajouter fonction service :

validateSemesterCredits(programId, levelId, semesterId)
