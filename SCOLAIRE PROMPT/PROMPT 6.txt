PROMPT 6

 — Inscriptions Administratives (IA) + Facturation initiale + Attestation PDF + UI Scolarité “dernier cri”

Objectif : Implémenter l’Inscription Administrative (IA) complète (backend + frontend), liée à la finance (facture initiale), avec règles strictes : unicité, statut, validation, audit, et génération automatique d’Attestation d’Inscription / Scolarité PDF.

Dépend des Prompts 3 (référentiel), 5 (students), 2 (auth/audit), et prépare le Prompt 7 (IP).

0) RBAC

IA CRUD + validation : SCOLARITE, ADMIN, SUPER_ADMIN

Consultation : AUDITOR, DEPT_HEAD (sur périmètre département/programmes)

Accès étudiant : seulement “voir son IA” (endpoints me) dans un prompt futur

Audit obligatoire sur toutes actions.

2) Backend — Règles métier IA
2.1 Workflow IA

DRAFT : créé par scolarité (brouillon)

PENDING : dossier soumis (optionnel)

VALIDATED : validée par scolarité

REJECTED : rejet avec motif

CANCELED : annulation (audit + rôle)

2.1.1 Règles

Un étudiant ne peut pas avoir 2 IA pour la même année (unicité)

Si IA VALIDATED : impossible de modifier programme/niveau (sauf procédure “rectification IA” option future)

IA bloquée si année CLOSED

IA bloquée si étudiant DISABLED/SUSPENDED selon règles

Au passage VALIDATED :

générer attestation PDF

générer facture (si paramétrage frais IA)

audit log

3) API REST — Endpoints IA

Base /api/v1

3.1 IA

GET /enrolments/ia?page=&pageSize=&q=&academicYearId=&programId=&levelId=&status=

POST /enrolments/ia

GET /enrolments/ia/:id

PATCH /enrolments/ia/:id (seulement si DRAFT/PENDING)

POST /enrolments/ia/:id/submit (DRAFT -> PENDING)

POST /enrolments/ia/:id/validate (-> VALIDATED)

Body : { "generateInvoice": true } (default true)

POST /enrolments/ia/:id/reject (-> REJECTED) body { reason }

POST /enrolments/ia/:id/cancel (-> CANCELED) body { reason }

3.2 Attestation IA

GET /documents/ia/:enrollmentIaId (métadonnées + lien)

POST /documents/ia/:enrollmentIaId/regenerate (version +1)

GET /documents/download/:fileId (signed URL)

GET /documents/verify?token=... (public : retourne validité + info minimale)

Info minimale verify :

nom, matricule, année, programme, statut VALIDATED (oui/non)

pas d’email/tel/DOB
