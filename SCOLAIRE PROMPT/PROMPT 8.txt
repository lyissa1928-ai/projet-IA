PROMPT 8
 — Emploi du Temps (EDT) + Détection Conflits + Vues Calendrier + Export PDF + Portails Étudiant/Enseignant

Objectif : Implémenter la gestion complète de l’emploi du temps (backend + frontend) :

Création de séances (CM/TD/TP)

Affectation : UE + groupe + enseignant + salle

Détection automatique des conflits (salle/enseignant/groupe)

Vues calendrier modernes (semaine/jour)

Export PDF (EDT groupe / enseignant)

Portail enseignant : “mes séances”

Portail étudiant : “EDT de mon groupe”

Dépend de Prompts 3-4 (référentiel + UE + groupes + salles), 5 (enseignants/étudiants), 7 (IP utile mais pas strict), 2 (auth/audit).

1) Modèle de données (Prisma)
1.1 TimetableSession

Créer table :

id

courseUnitId (FK CourseUnit) obligatoire

groupId (FK Group) obligatoire

teacherId (FK Teacher) obligatoire

roomId (FK Room) obligatoire

semesterId (FK Semester) obligatoire

startAt (datetime)

endAt (datetime)

type enum: CM | TD | TP

status enum: SCHEDULED | CANCELED | POSTPONED

notes (string nullable)

createdAt, updatedAt

Contraintes :

startAt < endAt

Durée max configurable (option)

courseUnit.semesterId == semesterId (cohérence)

group doit correspondre program/level de l’UE (warning ou blocage)

1.2 TimetableConflictLog (optionnel mais utile)

Créer table :

id

sessionId nullable (si tentative création)

conflictType enum: ROOM | TEACHER | GROUP

conflictWithSessionId

details JSON

createdAt

2) Backend — Détection Conflits (obligatoire)

Lors création ou modification d’une séance, vérifier chevauchement :

Deux séances A et B sont en conflit si :

A.startAt < B.endAt AND A.endAt > B.startAt

Conflits à détecter :

même roomId

même teacherId

même groupId

Réponse API en cas conflit :

code TIMETABLE_CONFLICT

details : liste conflits { type, sessionId, startAt, endAt, room/teacher/group }

Prévoir mode force=true (admin only) pour override (avec audit + log).
