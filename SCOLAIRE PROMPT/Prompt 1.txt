Prompt 1 


✅ PROMPT 1 — Bootstrap Fullstack (Web) “Dernière génération”

Objectif : Initialiser un monorepo fullstack prêt production : Backend NestJS + PostgreSQL + Prisma et Frontend Next.js (App Router) + Tailwind + shadcn/ui, avec Docker, CI, Swagger, conventions, qualité, thèmes UI modernes, pages de base, layout, navigation, auth placeholder.

1) Stack imposée

Backend

NestJS (TypeScript)

Prisma ORM

PostgreSQL

Swagger OpenAPI auto

Validation DTO (class-validator / class-transformer)

Logger structuré (pino ou nestjs-pino)

Config env (dotenv + validation)

Tests (Jest)

Frontend

Next.js (App Router) TypeScript

TailwindCSS

shadcn/ui (Radix)

Icons lucide-react

React Query (TanStack Query) pour API calls

Zod + react-hook-form

Dark mode + theme toggle

Responsive design + animations légères (framer-motion)

DevOps

Docker Compose (postgres + api + web)

migrations automatisées

CI (GitHub Actions) : lint + tests + build

Prettier + ESLint + editorconfig

2) Monorepo & structure

Créer un monorepo avec cette structure :

sis/
  apps/
    api/         (NestJS)
    web/         (Next.js)
  packages/
    shared/      (types, zod schemas, utils)
    ui/          (optionnel si besoin)
  infra/
    docker/
  docs/
  .github/workflows/
  docker-compose.yml
  README.md

Utiliser pnpm workspaces (ou npm workspaces) pour gérer le monorepo.

Ajouter scripts root : dev, build, lint, test, db:up, db:migrate, db:seed.

3) Backend — détails à implémenter maintenant
3.1 Configuration & qualité

@nestjs/config + validation env (zod ou joi)

Structure modules : app.module.ts + health, common, config, database

Logger structuré (pino) + request-id

CORS configuré pour le frontend

Global pipes : ValidationPipe (transform, whitelist)

Global exception filter : erreurs standard JSON :

{ "code": "VALIDATION_ERROR", "message": "...", "details": {...}, "requestId": "..." }
3.2 Prisma + DB

Prisma schema initial + migrations

Tables initiales minimalistes :

User (id uuid, matricule unique, email unique, phone, passwordHash, role, status, firstLoginRequired, createdAt, updatedAt)

AuditLog (id, actorUserId nullable, action, entity, entityId, before JSON, after JSON, ipAddress, userAgent, createdAt)

Seed script : créer un Super Admin:

matricule: ADMIN0001

email: admin@univ.local

phone: 770000000

password = ADMIN0001 hashé

role = SUPER_ADMIN

firstLoginRequired = true

3.4 Swagger

Swagger accessible : /api/docs

API base : /api/v1
