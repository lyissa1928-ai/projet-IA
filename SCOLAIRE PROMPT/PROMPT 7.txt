PROMPT 7 
— Inscription Pédagogique (IP) + Choix UE + Règles crédits + Blocages (impayés/prérequis) + UI Scolarité & Portail Étudiant

Objectif : Implémenter l’Inscription Pédagogique (IP) complète (backend + frontend) par semestre, avec :

Précondition IA validée

Sélection UE (obligatoires + optionnelles)

Contrôle total crédits

Contrôle prérequis (optionnel mais prévu)

Blocage impayés (selon paramétrage)

Génération “carte / état IP” et logs d’audit

UI scolarité (gestion IP) + UI étudiant (voir état IP)

Dépend des Prompts 3 (référentiel), 4 (UE), 6 (IA + facture), 5 (students), 2 (auth/audit).

0) RBAC

CRUD IP + validation : SCOLARITE, ADMIN, SUPER_ADMIN

Consultation : DEPT_HEAD, AUDITOR

Étudiant : accès à son IP en lecture (endpoints me)

Audit obligatoire.

1) Modèle de données (Prisma)
1.1 EnrollmentIP

id

Contraintes :

Unicité (studentId, semesterId) unique

Semester doit appartenir à l’année de l’IA active de l’étudiant

Interdire IP si IA non VALIDATED pour même année

Interdire IP si semester.status=CLOSED ou année CLOSED

2) Backend — Logique métier IP
2.1 Préconditions IP

Pour créer/valider une IP :

L’étudiant doit avoir une IA VALIDATED pour l’année académique du semestre

L’étudiant doit être ACTIVE

Semestre OPEN

Si FinanceRule.blockIPIfUnpaid=true :

Vérifier solde facture (Invoice.totalAmount - paidAmount)

Si solde > threshold => BLOQUER IP

2.2 Sélection des UE

Le système doit :

Proposer automatiquement les UE obligatoires (selon program+level+semester)

Permettre ajout d’UE optionnelles (si l’université en a)

Supporter UE mutualisées

2.3 Contrôle crédits

Total crédits IP = somme crédits des UE sélectionnées

Seuil recommandé : 30 crédits / semestre (configurable)

Si total hors plage autorisée :

en DRAFT : warning

en VALIDATION : blocage (sauf override admin option)

2.4 Contrôle prérequis (si activé)

Si UE a prérequis :

vérifier que l’étudiant a validé l’UE prérequise (on utilisera notes plus tard)
Pour ce prompt, prévoir l’API et laisser vérification en “soft check” (warning) si notes pas encore.
Ou créer table “ValidatedCourseUnit” temporaire (option) si université veut.
